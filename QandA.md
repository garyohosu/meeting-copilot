## Q1. Pixel Recorder 共有URLは認証なしで取得できるか？

**該当箇所:** §7.1-A, §F-003, §15

**回答:**
Phase 1 では「認証なしで取得できるURLのみ対応」とし、本文貼り付けを主手段にする。
Pixel Recorder 共有URLの自動取得は補助機能に下げる。

**仕様反映:**
- F-003: 認証不要URLのみ取得対象、認証が必要な場合は取得失敗として扱い本文貼り付けへ誘導
- §15: Pixel Recorder のURL仕様変更や認証要否により取得不能となる場合がある旨を追記
- PoC成功条件は本文貼り付けで価値が出ること

## Q2. 入力サイズ上限の具体値は？

**該当箇所:** §7.3「入力サイズ上限を超えないこと」

**回答:**
- 文字数上限: 50,000文字
- 推奨文字数: 20,000文字以内
- 上限超過時: 自動分割せずエラー表示
- 将来拡張で分割処理対応

**仕様反映:**
- §7.3 に上限値と推奨値を明記
- エラー文言例: 「入力文字数が上限を超えています。50,000文字以下にしてください」

## Q12. セッション中のデータはどう扱うか？

**該当箇所:** §10.4「入力された文字起こし内容を不用意に外部保存しないこと」

**回答:**
- サーバー側には永続保存しない
- 処理中メモリ上では保持
- ブラウザ再読み込みで消えてよい
- localStorage 一時保存はデフォルトOFF
- ログにはエラー概要のみ記録し、本文は含めない

**仕様反映:**
- §10.4 に明記

## Q14. OpenAI API への送信をどう位置づけるか？

**該当箇所:** §10.4「外部保存しない」, §16.1「OpenAI API を使用」

**回答:**
本アプリは OpenAI API を利用するため、入力された文字起こし本文および追加指示は OpenAI API に送信される。
ただし、アプリ自身はサーバー側に本文・生成結果を永続保存しない。
機密情報を含む会議データを扱う場合は、利用者が送信可否を判断することを前提とする。

**仕様反映:**
- §10.4 に「文字起こし本文・追加指示・再生成指示は OpenAI API に送信される」ことを明記
- §10.4 に「アプリ自身はサーバー側に永続保存しない」ことを明記
- §10.4 に機密情報・個人情報を含む会議データの送信可否確認を明記
- §15 に「外部送信が許可されない会議データには適用できない場合がある」を追記

## Q15. 追加指示は初回生成用と再生成用でどう分けるか？

**該当箇所:** §7.2 任意項目「追加指示」, §11.2「タブごとに追加指示欄付き」, §F-010

**回答:**
初回生成では、メイン画面の全体共通の追加指示を使用する。
再生成では、初回の全体追加指示を保持したまま、タブごとの再生成指示を追記して使用する。
メイン画面には「追加指示」入力欄を明記する。

**仕様反映:**
- §7.2 の追加指示を「初回生成用・全体共通」と明確化
- §11.1 に「追加指示入力欄（初回生成用・全体共通）」を追加
- F-010 に「再生成時は初回生成時の全体追加指示に加えて、対象タブ専用の追加指示を適用できること」を追記
